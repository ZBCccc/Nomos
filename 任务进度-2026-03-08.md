```

├─────────────────────────────────────────────────────────────┤
│ query → a_j = H(w_j)^{r_j}                                  │
│         b_j = H(w||j||0)^{s_j}                              │

                            ↓
┌─────────────────────────────────────────────────────────────┐

│ bstag'_j = b_j^{K_T[I1] * gamma_j}                         │
│ delta'_j = c_j^{K_T[I1]}                                   │
│ bxtrap'_j = a_j^{K_X[Ij] * rho_j}                         │
│ env = Enc_{K_M}(rho, gamma)                                │
└─────────────────────────────────────────────────────────────┘
                            ↓
┌─────────────────────────────────────────────────────────────┐

└─────────────────────────────────────────────────────────────┘
                            ↓
┌─────────────────────────────────────────────────────────────┐
```

## 技术要点

### 1. 为什么 OPRF token ≠ 简化 token？

**简化版本**:
```
bstag_j = H(w||j||0)^{K_T[I1]}
```

**OPRF 版本**:
```
Client 去盲化后: bstag_j = H(w||j||0)^{K_T[I1] * gamma_j}
Server 去盲化后: stag_j = bstag_j^{gamma_j^{-1}} = H(w||j||0)^{K_T[I1]}
```

**关键点**: OPRF token 需要 Server 端额外去盲化才能匹配 TSet 中的 stag。这是论文设计的一部分，gamma 用于随机化和防止 token 重放。

### 2. env 的作用

**内容**: `env = Enc_{K_M}(rho_1..n, gamma_1..m)`

**用途**:
- Server 需要 gamma 来去盲化 stag
- Server 需要 rho 来去盲化 xtag
- 加密保证只有持有 K_M 的 Server 可以解密

**当前实现**: XOR 加密（简化）
**建议**: 升级为 AES-GCM（AEAD）

### 3. 内存管理

**RELIC 类型特点**:
- `bn_t` 和 `ep_t` 是数组类型（`bn_st[1]`, `ep_st[1]`）
- 不能直接用 `std::vector<bn_t>`
- 必须使用手动内存管理

**正确模式**:
```cpp
bn_t* array = new bn_t[n];
for (int i = 0; i < n; ++i) {
    bn_new(array[i]);  // 初始化
    // ... 使用 ...
    bn_free(array[i]);  // 释放
}
delete[] array;
```
